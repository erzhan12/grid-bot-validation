# 0017 — Remove asyncio from gridbot: bbu2-style polling main loop

## Description

Replace all `asyncio` usage in `apps/gridbot` with a single-threaded sequential
polling loop modelled on `bbu2-master/controller.py:check_job()`. WebSocket
callbacks (running in pybit's internal threads) only write to shared caches /
deques. The main thread runs a `while self._running: tick(); time.sleep(0.1)`
loop that drains pending events, processes the latest ticker per symbol, runs
periodic checks via timestamp gating, and ticks retry queues inline. No
`queue.Queue`, no dispatcher, no background threads, no locks.

The only concurrency remaining is pybit's internal WS threading — invisible to
business logic and limited to "write to a deque / overwrite a dict field".

**Rationale**: current asyncio implementation serialises everything onto one
event loop anyway (no `asyncio.gather` in gridbot), so there is no actual
parallelism to lose. The refactor makes the existing sequentiality explicit
and breakpoint-friendly, matching the bbu2 mental model the author already
understands.

    config = load_config(...)

    orchestrator = Orchestrator(config=config, db=db, ...)

**New `def run()` — the main polling loop** (blocking):
```python
def run(self) -> None:
    """Main polling loop. Blocks until request_stop() / stop()."""
    logger.info("Orchestrator main loop started")
    while self._running:
        try:
            self._tick()
        except Exception as e:
            self._notifier.alert_exception("main_loop", e, error_key="main_loop")
        time.sleep(CHECK_INTERVAL)
    logger.info("Orchestrator main loop exited")

def _tick(self) -> None:
    # 1. Drain pending execution events
    for runner in self._runners.values():
        dq = self._pending_executions.get(runner.strat_id)
        if dq:
            while True:
                try:
                    event = dq.popleft()
                except IndexError:
                    break
                runner.on_execution(event)

    # 2. Drain pending order-update events
    for runner in self._runners.values():
        dq = self._pending_orders.get(runner.strat_id)
        if dq:
            while True:
                try:
                    event = dq.popleft()
                except IndexError:
                    break
                runner.on_order_update(event)

    # 3. Process latest ticker per symbol (coalesced — free with overwrite)
    for symbol, runners in self._symbol_to_runners.items():
        event = self._latest_ticker.get(symbol)
        if event is None or event is self._last_processed_ticker.get(symbol):
            continue
        self._last_processed_ticker[symbol] = event
        for runner in runners:
            runner.on_ticker(event)

    # 4. Periodic checks via timestamp gating (bbu2 pattern)
    now = time.monotonic()
    if now >= self._next_position_check:
        self._fetch_and_update_positions()
        self._next_position_check = now + self._config.position_check_interval
    if now >= self._next_health_check:
        self._health_check_once()
        self._next_health_check = now + _HEALTH_CHECK_INTERVAL
    if now >= self._next_order_sync and self._config.order_sync_interval > 0:
        self._order_sync_once()
        self._next_order_sync = now + self._config.order_sync_interval
    if now >= self._next_retry_tick:
        for rq in self._retry_queues.values():
            rq.process_due()
        self._next_retry_tick = now + 1.0
```

Notes:
- `_health_check_once()` and `_order_sync_once()` are extractions of the body
  of the existing `_health_check_loop` / `_order_sync_loop` — the `while` + `sleep`
  parts are removed, the inside becomes a single-shot function.
- Exceptions inside `_tick()` are caught at the top of `run()` so one bad
  iteration does not kill the loop (matches bbu2 `check_job` pattern).

**`def request_stop()`** (new, tiny method — for signal handlers):
```python
def request_stop(self) -> None:
    self._running = False
```

**`stop()` → `def stop()`** (cleanup, non-blocking):
- `self._running = False` (idempotent)
- WS disconnects (already sync)
- Remove retry queue `stop()` calls — no thread to stop; final `process_due` pass is optional
- **Delete** embedded EventSaver stop block
- `await self._update_run_records_stopped()` → sync
- No task joins (no tasks)

**WS callback rewrite — these are the entire bridge between pybit threads and main thread**:

```python
def _on_ticker(self, account_name: str, symbol: str, message: dict) -> None:
    try:
        normalizer = self._normalizers[account_name]
        event = normalizer.normalize_ticker(message)
        if event is not None:
            self._latest_ticker[symbol] = event   # atomic dict-set under GIL
    except Exception as e:
        self._notifier.alert_exception("_on_ticker", e, error_key="ws_on_ticker")

def _on_position(self, account_name: str, message: dict) -> None:
    try:
        if account_name not in self._position_ws_data:
            self._position_ws_data[account_name] = {}
        for pos in message.get("data", []):
            if pos.get("category") != "linear":
                continue
            symbol = pos.get("symbol", "")
            side = pos.get("side", "")
            if not symbol or not side:
                continue
            self._position_ws_data[account_name].setdefault(symbol, {})[side] = pos
    except Exception as e:
        self._notifier.alert_exception("_on_position", e, error_key="ws_on_position")

def _on_order(self, account_name: str, message: dict) -> None:
    try:
        normalizer = self._normalizers[account_name]
        events = normalizer.normalize_order(message)
        for event in events:
            for runner in self._symbol_to_runners.get(event.symbol, []):
                if self._get_account_for_strategy(runner.strat_id) == account_name:
                    self._pending_orders[runner.strat_id].append(event)   # deque.append is atomic
    except Exception as e:
        self._notifier.alert_exception("_on_order", e, error_key="ws_on_order")

def _on_execution(self, account_name: str, message: dict) -> None:
    try:
        normalizer = self._normalizers[account_name]
        events = normalizer.normalize_execution(message)
        for event in events:
            for runner in self._symbol_to_runners.get(event.symbol, []):
                if self._get_account_for_strategy(runner.strat_id) == account_name:
                    self._pending_executions[runner.strat_id].append(event)
    except Exception as e:
        self._notifier.alert_exception("_on_execution", e, error_key="ws_on_execution")
```

Thread safety rationale:
- `dict[k] = v` is atomic under CPython GIL (single STORE_SUBSCR bytecode).
- `deque.append` / `deque.popleft` are atomic under CPython GIL.
- `_latest_ticker[symbol] = event` racing with main-loop read: worst case is
  main loop reads an event that has already been replaced — acceptable, the
  next tick will pick up the newer one.
- No explicit locks anywhere.

**REST wrapper methods** (all `async def` → `def`, remove `asyncio.to_thread` / `asyncio.wait_for`):

- `_fetch_wallet_balance()`: `await asyncio.to_thread(rest_client.get_wallet_balance)` → `rest_client.get_wallet_balance()`

- `_get_wallet_balance()`: remove `async with self._wallet_cache_lock` (lock removed); call `self._fetch_wallet_balance(account_name)` directly

## Algorithm: new sequential main loop

```
cli()
  └─ main()                                    ← sync, no asyncio.run
       ├─ load config
       ├─ init DB
       ├─ orchestrator = Orchestrator(...)
       ├─ register signal handlers → orchestrator.request_stop()
       ├─ orchestrator.start()                 ← init (non-blocking)
       │    ├─ init accounts (REST, WS clients, normalizers)
       │    ├─ init strategies (runners, executors, retry queues)
       │    ├─ startup reconciliation (sync REST)
       │    ├─ create DB run records
       │    ├─ connect WS (pybit internal threads start)
       │    ├─ fetch initial positions (sync REST)
       │    └─ seed _latest_ticker / deques, set _running = True
       │
       ├─ orchestrator.run()                   ← BLOCKS (polling loop)
       │    while self._running:
       │      _tick():
       │        drain pending executions  → runner.on_execution()
       │        drain pending orders      → runner.on_order_update()
       │        process latest tickers    → runner.on_ticker()
       │        time-gated periodic calls (pos, health, order_sync, retry)
       │      time.sleep(0.1)
       │
       └─ orchestrator.stop()                  ← on SIGINT/SIGTERM (via finally)
            disconnect WS
            update Run records
            (retry queues have no thread — nothing to join)
```

WS thread → cache/deque → main thread:

```
pybit WS thread                           main thread (every 100 ms)
────────────────────                      ───────────────────────────
_on_ticker(...)                           _latest_ticker[symbol] → runner.on_ticker(event)
   └─ _latest_ticker[symbol] = event            └─ engine.on_event(event, limit_orders)
                                                     └─ executor.execute_place(intent)

_on_execution(...)                        _pending_executions[sid].popleft() → runner.on_execution(event)
   └─ _pending_executions[sid]                  └─ engine.on_event(event)
           .append(event)                            └─ executor.execute_place(intent)

_on_position(...)
   └─ _position_ws_data[acc][sym][side]
           = pos   (overwrite)
```

All `async def test_*` / `@pytest.mark.asyncio` tests in `apps/gridbot/tests/`
become plain `def test_*`. Helpers that `await runner.on_ticker(...)` become
direct calls. Mock patches on `asyncio.run_coroutine_threadsafe` are removed.
Instead, tests inspect `orchestrator._latest_ticker`, `_pending_executions`,
`_pending_orders` directly, or drive `orchestrator._tick()` manually.

Affected test files:
- `apps/gridbot/tests/test_runner.py`
- `apps/gridbot/tests/test_orchestrator.py` — also remove embedded-EventSaver tests
- `apps/gridbot/tests/test_retry_queue.py` — remove `start`/`stop`/`_run_loop` tests; keep `process_due`, `add`, `clear`
- `apps/gridbot/tests/test_reconciler.py`
- `apps/gridbot/tests/test_main.py` — remove `--save-events` CLI test
- `tests/integration/test_runner_lifecycle.py`
- `tests/integration/test_eventsaver_db.py` — verify it tests standalone EventSaver (untouched). If it tests embedded mode, delete those cases.

## Out of scope (deferred)

### 0018 — Port standalone EventSaver from asyncio to sync (optional)

In 0017 we remove the embedded EventSaver mode from gridbot but leave the
standalone `apps/event_saver` service on asyncio. Deferring the sync port is
intentional: EventSaver is I/O-heavy persistence (bulk DB inserts) where
asyncio is a natural fit, and debugging is not painful because there is no
strategy/intent logic — only buffer → flush.

Port to sync only if:
- single-codebase uniformity becomes more valuable than leaving it alone, or
- actual debugging pain emerges in standalone EventSaver.

Estimated scope: ~2.6k lines across 13 files. Pattern would mirror 0017:
polling loop, time-gated auto-flush, `threading.Lock` on writer buffers
(since WS callbacks write from pybit threads while flush runs on the main
thread).

### Deploy note (effective immediately after 0017)

Running `apps/gridbot` no longer optionally starts EventSaver. If event
capture is required in a deployment, run `apps/event_saver` as a separate
process (its own `uv run python -m event_saver.main`). Update docker-compose
/ systemd / supervisord configs accordingly.
