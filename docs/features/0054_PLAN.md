# Feature 0054 — Fail-Loud on Missing Grid State When `seed.enabled=true`

## Context

When `seed.enabled=true` and no `grid_state` can be loaded from either DB or file, the engine
silently falls back to `grid_source="fresh"` and proceeds with a from-scratch grid. This hid the
0052 bug for 3 days: every replay logged `grid seed source=file` (or `fresh`) with no exception,
and v8 validation's 100% match rate was held together only by manual JSON file copies before each
run.

**Rule:** grid state is hard-required when `seed.enabled=true`. If both the DB loader and the file
loader return `None` (including step/count mismatch — loaders treat mismatch the same as absence),
raise `SeedDataQualityError` immediately. Wallet is soft-required: when `load_wallet_seed_full`
returns `None`, the engine falls back to `initial_balance` and must emit a `WARNING`. Position and
orders loaders never return `None` (`load_position_snapshots` always returns a pair, including
`(zero, zero)`; `load_active_orders` returns `[]` for a clean account), so they are out of scope
for the soft-required warning.

When `seed.enabled=false` the current silent fresh-build behavior is unchanged.

## Algorithm — grid guard in `_load_seed`

Current flow after both loaders (inside the `with self._db.get_session()` block):

```
grid_seed = load_grid_state_from_snapshots(...)   # DB path
grid_source = "db"
if grid_seed is None and seed.grid_state_path is not None:
    grid_seed = load_grid_state(GridStateStore(...), ...)
    grid_source = "file" if grid_seed is not None else "fresh"
elif grid_seed is None:
    grid_source = "fresh"
logger.info("%s: grid seed source=%s", seed.strat_id, grid_source)
```

New flow:

```
grid_source = "db"
grid_seed = load_grid_state_from_snapshots(...)   # DB path
if grid_seed is None and seed.grid_state_path is not None:
    grid_seed = load_grid_state(GridStateStore(...), ...)
    grid_source = "file" if grid_seed is not None else None
elif grid_seed is None:
    grid_source = None

if grid_source is None:
    raise SeedDataQualityError(
        f"seed.enabled=True but no grid state found for "
        f"strat_id={seed.strat_id!r} account_id={seed.account_id!r} "
        f"symbol={config.symbol!r} at_ts={seed.at_ts.isoformat()} "
        f"grid_state_path={seed.grid_state_path!r}"
    )
logger.info("%s: grid seed source=%s", seed.strat_id, grid_source)
```

After the grid guard, existing loaders for position/wallet/orders run unchanged. After those
calls, **outside** the `with self._db.get_session()` context, add the wallet soft-required warning
(only `load_wallet_seed_full` can return `None`; position and orders loaders always return concrete
values):

```
if wallet_seed is None:
    logger.warning(
        "%s: wallet seed missing, defaulting to initial_balance",
        seed.strat_id,
    )
```

Loader contract reference (do not change loaders in this feature):

- `load_grid_state_from_snapshots` / `load_grid_state` → `None` on no row, legacy format, or
  step/count mismatch.
- `load_position_snapshots` → always `(PositionStateSeed, PositionStateSeed)`; one-sided rows
  raise `SeedDataQualityError` inside the loader.
- `load_wallet_seed_full` → `Optional[WalletSeed]`; `None` triggers engine fallback.
- `load_active_orders` → always `list[ActiveOrderSeed]`; `[]` is valid.
