# 0081 — Split `grid_db/repositories.py` into a `repositories/` package (issue #184, first extraction)

## Context

`shared/db/src/grid_db/repositories.py` is 1512 lines holding 13 repository classes
(`BaseRepository` + `T = TypeVar`, then 12 entity repositories). Issue #184 (P2) is an
incremental refactor: extract **ONE** concern with **ZERO** behavior change. This first
extraction converts the single module into a `repositories/` **package** of smaller,
domain-grouped modules, re-exported so that every existing import path keeps working
byte-for-byte at the call site.

Both public import paths are in active use and MUST keep working unchanged:

- `from grid_db import XRepository` — public API, re-exported in `grid_db/__init__.py:32-46`.
  Callers: `apps/comparator/{loader,equity}.py`, `apps/event_saver/.../writers/*.py` (7 files),
  `apps/gridbot/orchestrator.py`, `apps/replay/tests/test_collateral_mark_feed.py`,
  `scripts/verify_0065_collateral.py`, and many `from grid_db import …Repository` lines inside
  `shared/db/tests/test_repositories.py`.
- `from grid_db.repositories import XRepository` — used by `grid_db/__init__.py` itself,
  `shared/db/tests/test_repositories.py:6`, `apps/gridbot/.../writers/grid_state_writer.py`,
  `apps/gridbot/tests/test_orchestrator.py` (4 sites), `apps/gridbot/tests/test_grid_state_writer.py`,
  `apps/replay/tests/test_engine_seed.py`, `apps/replay/tests/test_snapshot_loader.py`.

Because `from grid_db.repositories import X` must keep resolving, the replacement MUST be a
package directory `grid_db/repositories/` with an `__init__.py` that re-exports every class —
NOT a renamed module. `shared/db/pyproject.toml` packages the whole `src/grid_db` dir
(`[tool.hatch.build.targets.wheel] packages = ["src/grid_db"]`), so the new subpackage is
auto-included; no packaging change needed.

## Design decisions

### 1. Layout: `repositories.py` → `repositories/` package

Delete the flat module; create `shared/db/src/grid_db/repositories/` with:

- `__init__.py` — re-exports every class + `BaseRepository`/`T`; defines `__all__`.
- `base.py` — `T = TypeVar("T", bound=Base)` and `BaseRepository`.
- per-domain modules (see grouping below), each importing `BaseRepository` from `.base`.

This preserves BOTH `from grid_db.repositories import X` (resolves against the package
`__init__`) AND `from grid_db import X` (no change needed to `grid_db/__init__.py`, which
imports from `grid_db.repositories`).

### 2. Grouping granularity: domain-grouped (4 modules), not one-per-class

Recommended — 4 domain modules instead of 12 single-class files. Justification: cohesion
(some classes already share helper imports like `func`/`tuple_`/`and_`/`or_` and the
dialect-insert helpers), avoids a sprawl of ~13 tiny files, and keeps each module small
(~150–400 lines). Cross-class coupling is minimal: the only inter-repository code reference
is `GridStateSnapshotRepository.get_at_or_before`/`get_active_run_at_or_before` joining the
`Run` **model** (imported from `grid_db.models`, not from `RunRepository`) — so no module
needs to import another repository module.

Module → classes (preserve current source order within each file):

- `base.py` — `T`, `BaseRepository`
- `identity.py` — `UserRepository`, `BybitAccountRepository`, `ApiCredentialRepository`,
  `StrategyRepository`, `RunRepository`
- `market_data.py` — `PublicTradeRepository`, `TickerSnapshotRepository`
- `execution.py` — `PrivateExecutionRepository`, `OrderRepository`
- `snapshots.py` — `PositionSnapshotRepository`, `WalletSnapshotRepository`,
  `GridStateSnapshotRepository`

Note: `identity.py` is a NEW module under `repositories/`; it does not collide with the
existing top-level `grid_db/identity.py` (different package level — `grid_db.repositories.identity`
vs `grid_db.identity`). Acceptable, but if reviewer prefers zero name overlap, rename to
`accounts.py`. Default: keep `identity.py`.

### 3. Re-export mechanism

`repositories/__init__.py` does explicit `from .base import BaseRepository, T` and
`from .<module> import <Classes>` for each, then sets `__all__` listing all 13 class names
(plus `BaseRepository`; include `T` only if currently importable — it is not in any caller,
so omit from `__all__` but still import it into the package namespace to match the old module
namespace where `repositories.T` existed). Keep the existing `grid_db/__init__.py` import
block and `__all__` UNCHANGED.

### 4. Per-module imports (zero behavior change)

Each module imports only what it uses (copy the relevant subset from the current top-of-file
import block):

- All modules: `from sqlalchemy.orm import Session`; `from .base import BaseRepository`
  (except `base.py`); the models each file references from `grid_db.models`.
- `base.py`: `from typing import Generic, TypeVar`; `from grid_db.models import Base`;
  `from sqlalchemy.orm import Session`.
- `identity.py`: models `User, BybitAccount, ApiCredential, Strategy, Run`;
  `from typing import Optional, List`; `from datetime import datetime`.
- `market_data.py`: models `PublicTrade, TickerSnapshot`; `from decimal import Decimal`;
  `from typing import Optional, List`; `from datetime import datetime`;
  `from sqlalchemy.dialects.sqlite import insert as sqlite_insert`;
  `from sqlalchemy.dialects.postgresql import insert as postgresql_insert`;
  `from sqlalchemy import insert`.
- `execution.py`: models `PrivateExecution, Order`; `from typing import Optional, List`;
  `from datetime import datetime`; `from sqlalchemy import func, tuple_`;
  dialect inserts (`sqlite_insert`, `postgresql_insert`, `insert`). NOTE: `OrderRepository.bulk_insert`
  has function-local `from sqlalchemy.dialects.postgresql import insert` / `...sqlite import insert`
  imports — keep those local imports verbatim (do NOT hoist; surgical move only).
- `snapshots.py`: models `PositionSnapshot, WalletSnapshot, GridStateSnapshot, Run`
  (Run needed for the `.join(Run, …)` in `GridStateSnapshotRepository`);
  `from typing import Optional, List`; `from datetime import datetime`;
  `from sqlalchemy import func, or_, and_, insert`; dialect inserts (`sqlite_insert`,
  `postgresql_insert`).

Class bodies, docstrings, method signatures, and source order are copied **verbatim** —
no edits to any line inside a class. This is a pure move + import partition + re-export shim.

## Zero-behavior-change verification (success criteria)

- `uv run pytest shared/db/tests -q` — all existing tests pass UNCHANGED (especially the
  90k-line `test_repositories.py`, which imports via BOTH paths). The new
  `test_repository_imports.py` passes.
- `uv run pytest apps/gridbot/tests/test_grid_state_writer.py apps/gridbot/tests/test_orchestrator.py apps/replay/tests/test_engine_seed.py apps/replay/tests/test_snapshot_loader.py -q`
  — downstream `from grid_db.repositories import GridStateSnapshotRepository` consumers still pass.
- `uv run ruff check .` — clean (watch for unused-import F401 in each new module: include ONLY
  the imports that module actually uses; this is the main lint risk of the split).
- Sanity import smoke (optional, fast): `uv run python -c "import grid_db; import grid_db.repositories; print('ok')"`.
- Diff discipline: `git diff` on the new files vs old should show only relocation + import
  partitioning + the re-export `__init__`; no diff inside any class body. A
  `git log --follow` / `git show` of the moved blocks should match the originals.

## Non-goals (do not do)

- No renaming for style; no changes to any method body, signature, or docstring.
- No big-bang split of other modules (`models.py`, `database.py`, etc.) — that is later #184 work.
- No change to `grid_db/__init__.py` or to any caller's import line.
- No hoisting of the function-local dialect imports in `OrderRepository.bulk_insert`.

## What could break (and the guarding test)

- `from grid_db.repositories import X` silently breaking if `repositories/__init__.py` misses a
  class → caught by `test_repository_imports.py` (dual-path + `is` identity over all 13 classes).
- An import landing in the wrong domain module (e.g. `func`/`tuple_`/`and_`/`or_`/dialect inserts)
  → caught by `ruff` (F821 undefined name) and by `test_repositories.py` exercising each method.
- Accidental duplicate model import or unused import → caught by `ruff` F401.
- `repositories.T` no longer importable for any niche caller → none found in grep; mitigated by
  importing `T` into the package namespace anyway.
