# Feature 0090 ŌĆö Source `tick_size` from exchange instead of hand-maintained YAML

## Context

`tick_size` is an exchange-defined instrument property (Bybit `priceFilter.tickSize`), but today it is a **required, hand-maintained YAML field** in three app configs. If Bybit revises an instrument's tick size, the YAML silently goes wrong and grid price rounding (`Grid._round_price`, `packages/gridcore/src/gridcore/grid.py:78-91`, driven by the YAML tick handed to `GridEngine`) starts producing invalid prices ŌåÆ order rejects.

The exchange value is *already fetched* in every app; the YAML is redundant:

- **gridbot** fetches live `InstrumentInfo` at startup (`apps/gridbot/src/gridbot/orchestrator.py:922` ŌåÆ `_fetch_instrument_info`, line 1158) but wires it to the runner only for qty rounding (`runner.py:1130-1145`) and the chase-order price/qty rounding (`runner.py:1967-1968`). **Grid** price rounding uses the YAML `tick_size` passed to `GridEngine` (`runner.py:445` ŌåÉ `config.py:113`, required).
- **replay/backtest** already have `InstrumentInfoProvider` (`apps/backtest/src/backtest/instrument_info.py`) ŌĆö Bybit public fetch + 24h local cache (`conf/instruments_cache.json`) + defaults. The replay engine builds it (`apps/replay/src/replay/engine.py:346`) and reads `qty_step`/`min_qty`/`max_qty` from it (line 876), but still passes the **YAML** `tick_size` into `BacktestStrategyConfig` (`engine.py:389` ŌåÉ `replay/config.py:30`, required).
- **gridcore** stays zero-dep by design: `GridEngine` receives `tick_size` as a `Decimal` parameter. **That contract does not change** ŌĆö only the *source* at the app layer.

The goal: **always take `tick_size` from the exchange**; make YAML `tick_size` an optional cross-check / what-if override.

### Scope decisions (from issue #222 open questions ŌĆö resolved here)

- **Mismatch policy (YAML override present):** gridbot = **fail-closed** (config drift = operator error worth surfacing); replay/backtest = **warn-and-use-YAML** (the override is intentional for what-if experiments). See phase details.
- **No-cache-no-network default trap:** `InstrumentInfoProvider.get()` currently returns a hardcoded `tick_size=0.1` default (`instrument_info.py:183-191`) ŌĆö a silent-wrong-value. Add a `require_live: bool = False` flag to `get()`; when True and neither API nor cache yields info, **raise** instead of returning defaults. This flag is for **replay/backtest only** ŌĆö gridbot does **not** call `InstrumentInfoProvider` (see below). Replay/backtest set it **dynamically**: `require_live=True` when YAML `tick_size` is `None` (the provider value becomes the engine tick ŌĆö defaults must never masquerade as exchange truth), `require_live=False` when a YAML override is present (YAML wins anyway; defaults then only affect the qty fields and mismatch-warning noise, preserving today's offline behavior).
- **gridbot does NOT use `InstrumentInfoProvider`:** gridbot's `_fetch_instrument_info` (`orchestrator.py:1174-1192`) calls `rest_client.get_instruments_info(symbol)` + `InstrumentInfo.from_bybit_response` **directly**, via the account-authenticated REST client ŌĆö no provider, no `pybit` `HTTP()`, no JSON cache. That path is deliberately kept (introducing the public `pybit` fetch and `conf/instruments_cache.json` into the live process is out of scope). gridbot's fail-closed semantics are realized by the retry+raise **inside `_fetch_instrument_info` itself**, NOT by `require_live=True` on the provider. `require_live` is therefore never exercised by gridbot.
- **Shared location:** gridbot must not import from `apps/backtest`. `InstrumentInfoProvider` moves to `packages/bybit_adapter` (already the Bybit-facing package; keeps gridcore dependency-free). Backtest/replay import from the new location; a thin re-export at the old path preserves existing imports.

## Phase 1 ŌĆö Move `InstrumentInfoProvider` to a shared package

Relocate the provider so all three apps can use it without cross-app imports.

- **Create** `packages/bybit_adapter/src/bybit_adapter/instrument_info.py` ŌĆö move the `InstrumentInfoProvider` class body from `apps/backtest/src/backtest/instrument_info.py` verbatim. It re-exports `gridcore.instrument_info.InstrumentInfo` (unchanged) and keeps the `pybit` fetch, the JSON cache (`conf/instruments_cache.json`), and the freshness logic. `bybit_adapter` already depends on `pybit`, so no new dependency is introduced.
- **Add `require_live` to `get()`** (`InstrumentInfoProvider.get(symbol, force_fetch=False, require_live=False)`):
  - Behavior unchanged for `require_live=False` (cache ŌåÆ API ŌåÆ cache ŌåÆ hardcoded defaults).
  - When `require_live=True` and the flow reaches the "no cache, use defaults" branch (`instrument_info.py:183-191`), raise `ValueError` (invalid instrument params, e.g. wrong tick, must not be silently defaulted) instead of returning the `tick_size=0.1` default.
  - **Stale-cache fallback stays allowed under `require_live=True`** (the API-failed ŌåÆ cached path at `instrument_info.py:177-181`): a stale cache entry is a *real* previously-fetched exchange value, not a synthetic default ŌĆö acceptable for offline replay/backtest determinism. `require_live` guards only the fabricated-defaults branch.
- **Keep `apps/backtest/src/backtest/instrument_info.py` as a thin re-export** (`from bybit_adapter.instrument_info import InstrumentInfoProvider, InstrumentInfo, DEFAULT_CACHE_PATH` ŌĆö `DEFAULT_CACHE_PATH` included because `apps/backtest/tests/test_instrument_info.py:10` imports it from the old path) so existing imports in `replay/engine.py:46` and `backtest/*` keep working with a one-line change or none. (This is not speculative back-compat; it avoids touching every import site in one PR.)
- **Fix silent-default parsing in `gridcore.InstrumentInfo.from_bybit_response`** (`packages/gridcore/src/gridcore/instrument_info.py:82-83`): missing `lotSizeFilter.qtyStep` / `priceFilter.tickSize` currently default to `"0.001"` / `"0.1"` ŌĆö positive values that pass the existing `<= 0` invalid-params check, so a malformed exchange response yields a *synthetic* tick accepted as exchange truth. Change: treat a **missing** `qtyStep` or `tickSize` key exactly like the invalid (`<= 0`) case ŌĆö log WARNING and return `None`. This affects all callers (gridbot direct path *and* the provider), which is intended: now that tick_size is load-bearing for price, no caller may receive a fabricated tick. (`min_qty`/`max_qty` defaults at lines 96-97 are qty-guardrails only and may stay.) No network/pybit import is added to gridcore ŌĆö this is pure dict parsing.
- Workspace deps: `apps/gridbot` already depends on `bybit-adapter` (`apps/gridbot/pyproject.toml:8,20`) ŌĆö no change. **`apps/backtest` does NOT** (`apps/backtest/pyproject.toml:6-13`: gridcore, grid-db, pyyaml, pydantic, pydantic-settings, pybit only) ŌĆö add `bybit-adapter` + the `[tool.uv.sources]` workspace entry, otherwise the re-export breaks installed backtest/replay imports outside the repo pytest `pythonpath`. Replay resolves through the backtest re-export, so the backtest dep covers it transitively.

## Phase 2 ŌĆö gridbot: use exchange `tick_size` for the grid engine (fail-closed)

- **`apps/gridbot/src/gridbot/config.py:113`** ŌĆö relax `tick_size` from required to `Optional[Decimal] = None`. **Extend `parse_tick_size` (`config.py:393-399`) to coerce `int`/`float` via `Decimal(str(v))`** (mirror replay's `_parse_decimal`, `replay/config.py:18-24`): today it converts only `str`, so an unquoted YAML `tick_size: 0.1` arrives as a float and `Decimal(0.1)`-style binary artifacts would make the exchange cross-check below spuriously mismatch and abort startup (the repo already carries unquoted values, e.g. `apps/replay/conf/replay.yaml:8`). `None` still passes through unchanged.
- **`apps/gridbot/src/gridbot/orchestrator.py:1158` (`_fetch_instrument_info`)** ŌĆö change from best-effort (returns `None` on failure) to **fail-closed**, mirroring the `_reconcile_startup_with_retry` pattern (`orchestrator.py:419`, feature 0086):
  - Retry transient fetch failures using a module-level backoff tuple (reuse the `_STARTUP_RECONCILE_BACKOFFS` style; define a sibling constant, do not inline magic values).
  - **Both failure modes count as a failed attempt**, not just exceptions: (a) the REST call raising, and (b) `InstrumentInfo.from_bybit_response` returning `None` (invalid or missing params ŌĆö today `orchestrator.py:1178-1183` soft-logs and returns `None` without raising). An exception-only retry wrapper would silently preserve the soft-`None` path and break the "always a valid `InstrumentInfo`" invariant. Uniform policy: `None` result ŌåÆ retry with backoff (a malformed payload can be a transient API glitch), exhaustion ŌåÆ abort as below.
  - On exhaustion: `self._notifier.alert(...)` then raise `StartupReconciliationError` (or a sibling startup exception) so `start()` aborts **before any order is placed** ŌĆö a wrong tick corrupts every grid level. This runs during `_init_strategy` at startup, before the main loop and WS connect, so blocking sleeps are safe (same rationale as 0086).
  - `_fetch_instrument_info` returning `None` for qty rounding must no longer be tolerated for a live strategy: with fail-closed fetch, `instrument_info` is always a valid `InstrumentInfo`.
- **`apps/gridbot/src/gridbot/orchestrator.py:922-958` (`_init_strategy` / runner construction)** ŌĆö the orchestrator **already** passes `instrument_info` to the runner (`orchestrator.py:940`); the runner stores it as `self._instrument_info`. **No new constructor param is needed here.** The only genuinely new orchestrator code in this block is (a) the fail-closed fetch call and (b) the mismatch cross-check (see below). Do not add a redundant `tick_size`/`instrument_info` constructor arg.
- **`apps/gridbot/src/gridbot/runner.py:443-450`** ŌĆö the source swap, **with a fallback because `GridEngine` is constructed inside `StrategyRunner.__init__`** (line 443) and `instrument_info` is an optional constructor param (`runner.py:204`, default `None`) that dozens of existing test constructions omit. Resolution at engine construction: `tick_size = self._instrument_info.tick_size if self._instrument_info is not None else strategy_config.tick_size`; if **both** are `None`, raise `ValueError` (no tick source). The live path always takes the exchange branch ŌĆö the orchestrator's fail-closed fetch guarantees a valid `InstrumentInfo` before the runner is built, and the orchestrator is the only production caller. The YAML branch exists for tests/tools that construct a bare runner; it is not a live fallback.
- **Cross-check (mismatch policy = fail-closed):** in `_init_strategy`, if `strategy_config.tick_size is not None` and it differs from `instrument_info.tick_size`, log ERROR and raise (config drift = operator error). If `None`, use exchange value silently. Document that the YAML field is being deprecated.

## Phase 3 ŌĆö replay/backtest: source `tick_size` from the provider (warn-and-override)

Determinism matters (`live_check`, feature 0088): the provider's 24h cache makes repeated runs offline-capable and stable.

- **`apps/replay/src/replay/config.py:30`** ŌĆö relax `tick_size` from required to `Optional[Decimal] = None`. Keep `parse_decimal_fields` validator (already no-ops on `None` ŌĆö confirm `_parse_decimal` returns `None` unchanged; it does).
- **`apps/replay/src/replay/engine.py:386-389`** ŌĆö the provider is already built (`engine.py:346`) and `instrument_info` fetched (`engine.py:876`). Change `BacktestStrategyConfig(..., tick_size=config.strategy.tick_size, ...)` to source from the provider's `InstrumentInfo.tick_size`.
  - **Ordering note:** `instrument_info` is currently fetched at `engine.py:876`, *after* `strategy_config` is built at `engine.py:386`. Move the provider `.get(config.symbol)` call earlier (before `BacktestStrategyConfig` construction) or reuse a single fetched instance for both the tick source and the later qty wiring ŌĆö one `.get()` call, used in both places.
  - **`require_live` is dynamic** (closes the default-trap loophole): call `.get(config.symbol, require_live=config.strategy.tick_size is None)`. When YAML tick is `None`, the provider value feeds the engine directly, so the no-cache-no-network `tick_size=0.1` default must **raise**, not silently become the grid tick. When a YAML override is present, `require_live=False` keeps replay fully offline-capable (YAML wins on mismatch anyway).
- **Mismatch policy = warn-and-use-YAML:** if `config.strategy.tick_size is not None` and it differs from the provider's value, log WARNING and **use the YAML value** (the what-if override wins for replay/backtest). If `None`, use provider value.
- **`apps/backtest/src/backtest/config.py:28`** ŌĆö relax `tick_size` to `Optional[Decimal] = None`. Its `parse_tick_size` validator (`config.py:99-104`) passes `None` through unchanged, but ŌĆö like gridbot's and unlike replay's `_parse_decimal` ŌĆö coerces only `str`: extend it with `int`/`float` ŌåÆ `Decimal(str(v))` so an unquoted YAML override tick doesn't produce float-artifact Decimals in the mismatch compare.
- **Backtest concretely builds a provider:** `InstrumentInfoProvider` is constructed at `engine.py:118`, `instrument_info` is fetched in `_init_runner` at `engine.py:244`, and the tick is consumed when `GridEngine` is constructed at `backtest/runner.py:172` (`tick_size=strategy_config.tick_size` ŌĆö exact mirror of replay/gridbot). Apply the same "provider value, YAML overrides with WARNING on mismatch" resolution before/at that `GridEngine` construction, reusing the already-fetched `instrument_info` instance (no extra `.get()`). The `engine.py:244` fetch uses the same dynamic flag on the **strategy** config's tick: `require_live=(strategy_config.tick_size is None)` (the enclosing `BacktestConfig` has no `tick_size` field ŌĆö it lives on the strategy config, `backtest/config.py:28`).

## Algorithm ŌĆö tick_size resolution (per app)

1. Fetch `InstrumentInfo` for the symbol (gridbot: direct `rest_client.get_instruments_info` in `_fetch_instrument_info`, fail-closed w/ retries+raise on exception **or** `None` ŌĆö no provider; replay/backtest: `InstrumentInfoProvider` w/ cache, `require_live = (yaml_tick is None)`).
2. `exchange_tick = info.tick_size`.
3. If YAML `tick_size` is `None`: `resolved = exchange_tick`.
4. If YAML `tick_size` is set and equals `exchange_tick`: `resolved = exchange_tick` (cross-check passed).
5. If YAML `tick_size` is set and differs:
   - gridbot ŌåÆ log ERROR, raise (fail-closed).
   - replay/backtest ŌåÆ log WARNING, `resolved = yaml_tick` (override wins).
6. Pass `resolved` to `GridEngine` / `BacktestStrategyConfig` (the gridcore contract is unchanged ŌĆö still a `Decimal` param).

## Testing (hermetic ŌĆö mock all network, per code-style)

- **gridcore `from_bybit_response`:** missing `priceFilter.tickSize` ŌåÆ `None`; missing `lotSizeFilter.qtyStep` ŌåÆ `None`; `tickSize <= 0` ŌåÆ `None` (existing); fully-populated payload unchanged.
- **Provider:** `require_live=True` + no cache + fetch fails ŌåÆ raises `ValueError` (no `0.1` default). `require_live=True` + API fail + **stale cache present** ŌåÆ returns the stale cached value (locks in the documented allow-stale decision). `require_live=False` no-cache path ŌåÆ returns default (byte-for-byte current behavior). Fresh-cache hit path unchanged.
- **gridbot orchestrator:** `_fetch_instrument_info`'s direct `rest_client.get_instruments_info` fails all retries ŌåÆ `StartupReconciliationError`, no orders placed, notifier alerted (mirror 0086 tests; mock the REST client, not the provider). Same abort when the REST call *succeeds* but returns a malformed payload (`from_bybit_response` ŌåÆ `None`) on every attempt ŌĆö and recovery when a later retry returns a valid payload; alert fires only on exhaustion. YAML `tick_size` present & matches ŌåÆ engine gets exchange tick. YAML present & mismatched ŌåÆ raises. YAML absent ŌåÆ engine gets exchange tick. Assert `GridEngine` receives `instrument_info.tick_size`, not the YAML value.
- **gridbot runner tests:** thanks to the Phase-2 fallback (`instrument_info.tick_size` if present, else `strategy_config.tick_size`), the many existing constructions that omit `instrument_info` (constructor default `None`, e.g. `test_runner.py:741-745`, `:1217-1221`, `:1398-1403`, `:4303`, `:4707-4714`) keep working unchanged ŌĆö their `strategy_config` fixtures carry a YAML tick. No mass fixture migration. New tests: engine gets `instrument_info.tick_size` when info is present (exchange wins over YAML); both-`None` ŌåÆ `ValueError`; `test_resolve_qty_no_instrument_info` (`:1398-1413`) stays valid as-is.
- **gridbot orchestrator test migration:** existing soft-`None` fetch tests (`test_orchestrator.py:3607-3639` style ŌĆö assert fetch failure is tolerated) must be rewritten for fail-closed semantics, not just supplemented.
- **replay engine:** provider tick used when YAML `None` (and `.get` called with `require_live=True`); YAML overrides with WARNING on mismatch (`require_live=False`); single `.get()` call (mock provider, assert call count). Determinism: two runs with a warm cache produce identical grid levels (guards `live_check`).
- **backtest engine/runner:** mirror the replay resolution cases at the `runner.py:172` `GridEngine` construction ŌĆö YAML `None` ŌåÆ provider tick; mismatch ŌåÆ WARNING + YAML wins; reuse of the `engine.py:244` fetched instance (no extra `.get()`).
- **Config models:** loading a YAML with no `tick_size` succeeds (all three configs); with `tick_size` present still parses to `Decimal`; unquoted numeric YAML (`tick_size: 0.1` as `float`, and `int`) coerces via `Decimal(str(v))` in the gridbot and backtest validators ŌĆö exact-equal to `Decimal("0.1")`, no fail-closed false mismatch.

## Risk

Low behavioral risk ŌĆö current YAML values (LTC 0.1, SOL 0.01) match the exchange, so grid levels are unchanged when the cross-check passes. Main risk is **startup availability** for gridbot, mitigated by fail-closed retries (same posture as #206/0086); replay/backtest are mitigated by the 24h cache.
