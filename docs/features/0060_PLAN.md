# 0060 â€” Fix position_value semantics mismatch (issue #141)

## Brief
The 0059 parity field `position_value` disagrees between live and backtest.
- **Live** (Bybit) stores `positionValue` verbatim = `|size| Ã— mark_price`.
- **Backtest** stores `|size| Ã— avg_entry_price`.
So the 0059 `pos_value_delta` is systematically nonzero whenever a position is
underwater or in-profit (delta â‰ˆ per-side unrealized PnL).

Fix: make the **backtest snapshot's reported `position_value`** mark-based so it
matches Bybit semantics. After the fix `pos_value_delta` â†’ ~0.

CRITICAL constraint (review finding 3): `calc_position_value(size, entry)` is NOT
a parity-only helper â€” its output also feeds **this project's local** margin
formulas (IM/MM/margin ratio via `_update_margin` / `calc_initial_margin`).
Bybit reports `positionValue` off **mark**; our local `calc_initial_margin` uses
**entry-based** notional (`RULES.md:2184` documents the known IM mismatch â€” Bybit
UTA cross-margin uses mark price and hedge optimization). Therefore the fix must
**split** the two values: local margin keeps entry-based; only the
**stored/snapshot** `position_value` becomes mark-based. Do NOT collapse them.

Also: correct the misleading `calc_position_value` docstring (it must NOT claim it
"matches Bybit positionValue"), update the stale model comment, add a regression
test, and add a guard test that IM/MM/margin ratio are unchanged.

  `abs(size) * mark_price` for the 0059 parity field, with
  `tracker.state.position_value` remaining entry-based for margin.

**mark==0 / unset guard (locked decision): NO guard.** Verified that
`mark_price` is always a real, positive price in the in-position branch:
- The same branch already passes `mark_price` into `calc_unrealised_pnl` (:655)
  and `_estimate_pair_im_mm` (:660), so the snapshot's `unrealised_pnl`,
  `position_im`, `position_mm`, and `mark_price` column already depend on it.

Why the emit site, not `_update_margin`: `_update_margin`'s result feeds IM/MM, so
mutating it would shift margin (review finding 3). The emit site is the single
chokepoint where the 0059 parity field is produced and the mark is already in scope.
