# Feature 0045 ‚Äî Hedge-aware IM/MM on `PositionSnapshot` for Bybit UTA parity

**Status:** PLAN
**Branch:** TBD ‚Äî create `feature/0045-hedge-aware-im-mm` before Phase 1 script work begins.
**Issue:** [#99](https://github.com/erzhan12/grid-bot-validation/issues/99)
**Prereqs:** 0034 (telemetry parity), 0042 (UTA totalAvailableBalance seed), 0043 (pair liquidation formula), 0044 (state-consistency filter) ‚Äî all shipped.

## Context

End-to-end Phase 4 validation on the 2026-05-17 mainnet recording closed
trade match (100%) and `liq_price_max_abs_delta` (0 USDT), but
per-leg `position_im` / `position_mm` on `PositionSnapshot` still
diverge from Bybit's reported values:

```
results/replay_ltcusdt_phase4_v3:
  position_im_max_abs_delta:  22.6 USDT
  position_mm_max_abs_delta:  1.77 USDT
```

The divergence has the same root cause that previously broke
`liq_price` (closed by 0043): per-leg
`calc_initial_margin(L_size*L_entry)` /
`calc_maintenance_margin(L_size*L_entry)` ignore the UTA hedge-mode
pooling rule. Bybit applies tier MMR to the **combined** notional and
distributes per-leg values back according to a rule we have not yet
characterised.

## Scope

- **In scope:** `apps/backtest/src/backtest/runner.py` snapshot emission
  path (`_emit_position_snapshot`, `_process_fill`), a new pair-aware
  IM/MM helper, tests, replay re-validation, docs.
- **Out of scope:** `gridcore.pnl.calc_initial_margin` /
  `calc_maintenance_margin` (correct as single-leg primitives, used by
  `pnl_checker` and other pure callers ‚Äî do NOT touch), live gridbot
  IM/MM reading (live reads `pos.positionIM` straight from Bybit, no
  computation), isolated margin mode (hedge mode only).

## Acceptance

- `position_im_max_abs_delta ‚â§ 1 USDT` and
  `position_mm_max_abs_delta ‚â§ 0.1 USDT` on the same Phase 4 dataset
  that previously showed 22.6 / 1.77.
- Long-only / short-only snapshots produce values **Decimal-equal** to
  the current per-leg path (zero leg collapses pair formula to single
  leg by construction). "Decimal-equal" means
  `helper_result == calc_initial_margin(L_pv)[0]` as `Decimal`
  comparison ‚Äî not byte-equality of downstream CSV / JSON
  representation, which depends on formatting and is out of scope.
- Unit suite covers long-only, short-only, balanced hedge, imbalanced
  hedge (both directions), zero positions, tier-boundary crossing.
- No changes outside `apps/backtest/`, `apps/replay/conf/`, `scripts/`,
  `docs/features/`, `RULES.md`, and corresponding tests.

## Phase 1 ‚Äî Derivation (research, no production code)

**Goal:** find a closed-form rule
`f(L_size, L_entry, S_size, S_entry, leverage, tier[, mark_price]) ‚Üí (im_long, im_short, mm_long, mm_short)`
that reproduces Bybit's live `position_im` / `position_mm` within
`max |Œî_IM| < 1 USDT`, `max |Œî_MM| < 0.1 USDT` across ‚â•10 paired
snapshots from Phase 4.

### Deliverable

- `scripts/research_0045_im_mm_distribution.py` ‚Äî read-only analytics
  script. No DB writes, no new tables.
- A markdown results table appended to this PLAN as **Phase 1 Results**
  (mirrors the format used in `0043_PLAN.md` Phase 2).

### Method

1. Load `position_snapshots` from Phase 4 DB (`source=live`). Pair
   long/short rows per pairing rule:
   - Primary: join by `(symbol, local_ts)` (recorder writes both legs
     from one REST `/v5/position/list` response, so `local_ts` is
     identical even when Bybit's per-leg `exchange_ts` drifts).
   - Fallback: for each long row, take the latest short row at or
     before its `local_ts`, and assert opposite-leg `size` / `entry`
     are stable across the window (no fill on the opposite side
     in-between). Drop the pair if stability check fails ‚Äî better to
     reject than to derive a rule from a moving target.
   - Reject pairs where either side's `size == 0` for the asymmetric
     hypotheses; keep them for the long-only / short-only sanity
     check (any hypothesis MUST collapse to H3 for zero-leg cases).
2. For each pair, compute:
   - `L_pv = L_size * L_entry`, `S_pv = S_size * S_entry`,
     `combined_pv = L_pv + S_pv`.
   - `combined_im, _ = calc_initial_margin(combined_pv, leverage, symbol, tiers)`.
   - `combined_mm, _ = calc_maintenance_margin(combined_pv, symbol, tiers)`.
   - Per-leg single-leg values (current production formula) for
     diff-vs-live baseline.
3. Tabulate columns:
   `ts | L_size | S_size | live_im_L | live_im_S | combined_im |
   per_leg_im_L | per_leg_im_S | ratio_L=live_im_L/combined_im |
   ratio_S=live_im_S/combined_im | weight_L=L_pv/combined_pv |
   weight_S=S_pv/combined_pv`.
   Mirror the MM block.
4. Score candidate distribution rules per pair and report `max |Œî|`
   per rule across the dataset:
   - **H1 notional-weighted (entry):**
     `X_leg = combined_X * leg_pv / combined_pv` with `leg_pv = size*entry`.
   - **H2 dominant-full + smaller-discounted:**
     `X_dominant = single_leg(dominant_pv);
      X_smaller  = combined_X - X_dominant`.
   - **H3 max-of-singletons:** `X_leg = single_leg(leg_pv)` (current
     production ‚Äî baseline; should not pass acceptance for hedge cases).
   - **H4 mark-notional variants of H1 / H2:** replace `leg_pv = size*entry`
     with `leg_pv_mark = size*mark_price`. UTA marginRatio is known to
     use mark, so this is plausible for IM/MM too.
   - **H5 dominant-pool with hedge-credited residual:**
     `combined_input = max(L_pv, S_pv) + Œ± * min(L_pv, S_pv)` for
     `Œ± ‚àà {0, 0.25, 0.5}` (typical hedge discount tiers), then
     distribute via H1 or H2.
   - **H6 direction-asymmetric:** compute long-side and short-side
     rules independently ‚Äî Bybit may publish the smaller leg's value
     with a hedge discount that does NOT mirror the larger leg's
     formula (matches the 0043 docstring observation about the
     smaller-leg discount being separate from the liq formula).
5. Edge cases checked explicitly:
   - `q_net = L_size - S_size == 0` (fully hedged).
   - `L_size == 0` or `S_size == 0` (collapses to single-leg; any
     winning rule MUST produce identical values to H3 here).

### Pass / fail rule

- A rule passes Phase 1 iff
  `max |Œî_IM| ‚â§ 1 USDT` AND `max |Œî_MM| ‚â§ 0.1 USDT` across the full
  paired dataset (no per-row exception).
- If multiple rules pass, prefer the simpler closed form.
- If none pass: STOP. Report to user. Possible escalations: add
  `mark_price` to inputs (Bybit may use mark, not entry, for hedge
  notional); add a fourth hypothesis driven by observed
  `ratio_L / weight_L` patterns. Do NOT proceed to Phase 2 with a rule
  that does not meet the threshold.

### Phase 1 exit artifact

Phase 1 ends with this PLAN updated in-place with:
- The winning rule's closed form.
- A table of residuals per snapshot for that rule.
- A one-paragraph rationale tying the result to Bybit's hedge-pooling
  semantics.

This artifact lands on `main` as a doc-only commit before Phase 2 work
starts (same shape as 0043 Phase 2 ‚Üí Phase 3 hand-off).

## Phase 2 ‚Äî Implementation

### New helper

Private method on `BacktestRunner` adjacent to
`_estimate_pair_liq_prices`:

```python
def _estimate_pair_im_mm(
    self,
    long_state,
    short_state,
) -> tuple[Decimal, Decimal, Decimal, Decimal]:
    """Hedge-aware pair IM/MM distribution.

    Returns ``(im_long, mm_long, im_short, mm_short)``. Mirrors
    ``_estimate_pair_liq_prices``: combined PV ‚Üí tier IM/MM ‚Üí distribute
    back per the formula derived in feature 0045 Phase 1.

    Long-only / short-only collapse to single-leg primitives by
    construction: the zero leg contributes zero to ``combined_pv``, so
    ``combined_im == single_leg(L_pv)`` and the smaller-side output
    is ``0``.
    """
```

Parameterised by `self._mm_tiers`, `self._leverage`, `self.symbol`
(same as `_estimate_pair_liq_prices`).

### Call-site change

`_emit_position_snapshot` (current `runner.py:647-668`) replaces:

```python
position_im, _imr = calc_initial_margin(position_value, self._leverage, self.symbol, tiers=self._mm_tiers)
position_mm, _mmr = calc_maintenance_margin(position_value, self.symbol, tiers=self._mm_tiers)
```

with an inline pair-aware call:

```python
im_long, mm_long, im_short, mm_short = self._estimate_pair_im_mm(
    self._long_tracker.state, self._short_tracker.state,
)
if direction == DirectionType.LONG:
    position_im, position_mm = im_long, mm_long
else:
    position_im, position_mm = im_short, mm_short
```

### No caller / signature change

Normal fill path calls `_emit_position_snapshot` from `_process_fill`
(`runner.py:602-605`); replay wind-down also calls it directly
(`engine.py:806` in `_wind_down`). Both paths emit **one snapshot per
call** ‚Äî for the just-mutated direction (fill path) or for each leg
sequentially (wind-down). There is no double-emit per fill, so the
precompute / kwargs-threading pattern used for `liq_long / liq_short`
in 0043 does NOT apply here. 0043's precompute is justified because
two consumers share the pair (`_update_risk_multipliers` at
`runner.py:584-588` AND `_emit_position_snapshot`); 0045 IM/MM has
exactly one consumer per emit, so inline call is the right shape.

This means `_emit_position_snapshot`'s signature does **not** gain
`im_long / mm_long / im_short / mm_short` kwargs. Every caller ‚Äî
`_process_fill` and `engine._wind_down` alike ‚Äî runs the same inline
pair-aware computation, no special casing.

### What does NOT change

- `_build_position_state` (`runner.py:686`) ‚Äî uses `calc_margin_ratio`,
  not IM/MM, formula correct.
- `gridcore.pnl.calc_initial_margin` / `calc_maintenance_margin` ‚Äî
  remain single-leg primitives, untouched.
- `apps/gridbot/` ‚Äî entire live trading path is out of scope. The new
  helper does not exist there and is not called from there.

### Failure mode

If the helper raises or returns `None` for any field, `_emit_position_snapshot`
**propagates the error**. No silent fallback to the per-leg formula:
divergent telemetry is the bug we are fixing, surfacing the failure is
safer than masking it.

## Tests

### Unit (`apps/backtest/tests/test_runner.py`, new `TestEstimatePairImMm`)

1. **Long-only** ‚Äî `S_size=0, L_size>0`: result equals
   `(calc_initial_margin(L_pv), calc_maintenance_margin(L_pv), 0, 0)`.
   Strict equality (zero tolerance).
2. **Short-only** ‚Äî symmetric.
3. **Balanced hedge** ‚Äî `L_size == S_size`, different entries.
   Hard-coded expected values taken from a real Phase 1 paired snapshot
   (golden fixture).
4. **Imbalanced hedge, long-dominant** ‚Äî `L_size > S_size > 0`. Asserts
   the per-leg ratio matches the rule derived in Phase 1 (exact
   formula filled in after Phase 1 closes).
5. **Imbalanced hedge, short-dominant** ‚Äî symmetric.
6. **Zero positions** ‚Äî `L_size = S_size = 0`: returns
   `(0, 0, 0, 0)`, no `DivisionByZero`.
7. **Tier-boundary** ‚Äî pair whose combined notional crosses the
   first LTCUSDT MMR tier (~50k USDT) while individual legs do not.
   Asserts combined-tier MMR is used, not sum-of-per-leg-tiers.

### Integration

8. `_emit_position_snapshot` long-direction call writes pair-aware
   `position_im` / `position_mm` into the returned snapshot.
9. `_emit_position_snapshot` short-direction call: same, symmetric.
10. `_process_fill` regression ‚Äî mock-counter on
    `_estimate_pair_im_mm`: exactly one helper call per fill,
    exactly one snapshot emitted (for the just-mutated direction).
    Guard against any future change that accidentally double-emits
    or recomputes.

### Golden fixture

Real `(L_size, L_entry, S_size, S_entry, live_im_long, live_mm_long,
live_im_short, live_mm_short)` row pulled from Phase 1 dataset. Stored
inline in test (no separate JSON file ‚Äî small enough). Tolerance
`‚â§ 0.1 USDT`.

## Risks

- **R1 ‚Äî No closed-form formula passes Phase 1.** Stop, surface to user.
  Possible escalation: add `mark_price` input, broaden hypothesis set.
- **R2 ‚Äî Formula depends on `mark_price` rather than entries.** Add
  `mark_price` to helper signature; `_emit_position_snapshot` already
  receives it, plumbing is trivial.
- **R3 ‚Äî Historical replay results inconsistent with new ones.** Do not
  rewrite old `results/replay_*` dirs. New replay writes to
  `replay_ltcusdt_phase4_v4/`. REVIEW doc captures before/after.
- **R4 ‚Äî Perf regression from inline call.** Each fill triggers one
  `_estimate_pair_im_mm` call. `_estimate_pair_liq_prices` already
  runs once per fill (different code path) without measurable cost;
  the IM/MM helper is the same shape and complexity. Mock-counter
  test #10 guards against accidental double-call.
- **R5 ‚Äî Divide-by-zero edges.** Two distinct cases:
  - `q_net == 0` (fully hedged, `L_size == S_size > 0`,
    `combined_pv > 0`) ‚Äî covered by balanced-hedge test #3.
  - `combined_pv == 0` (both legs zero) ‚Äî covered by zero-positions
    test #6. Helper has an explicit early-return guard for this case
    to avoid `DivisionByZero` regardless of distribution rule.
- **R6 ‚Äî Accidental change in `apps/gridbot/`.** Pre-merge file-list
  audit; scope is enforced in PR description and reviewer checklist.
- **R7 ‚Äî One-way mode (not hedge mode) regression.** One-way mode in
  backtest always presents as long-only OR short-only (no opposite
  leg). Tests #1 / #2 cover the collapse-to-single-leg case.

### Root causes ‚Äî why Phase 1 cannot conclude yet

**RC1 ‚Äî Tier rate mismatch (primary cause of all-fail).**
`MM_TIERS_DEFAULT` uses `(mmr=0.01, imr=0.02)` for the first tier, but
the data implies real LTCUSDT rates of roughly `mmr ‚âà 0.00715` (the
dominant leg's `live_mm_L / L_pv_mark` is **exactly** `0.00715` across
all rows ‚Äî `2.38048911 / 332.94 = 0.00715`) and `imr ‚âà 0.1` (a 10x
leverage tier, consistent with the `leverage: "10"` field in
`raw_json`). With wrong tier rates every hypothesis is comparing a
mis-scaled `combined_X` against the live values; the residuals are
dominated by the rate error, not the distribution rule.

**RC2 ‚Äî Insufficient state variability.**
Five snapshots, three unique states, identical leg sizes. We cannot
distinguish H1 vs H2 vs H6 from this data ‚Äî they all reduce to the
same residual when `L_size` and `S_size` are constant.

**RC3 ‚Äî Single-leg sanity check absent.**
The dataset has no `L_size == 0` or `S_size == 0` row, so we cannot
verify the collapse-to-single-leg behavior required by acceptance.

### Quantitative observations worth keeping

Independent of any distribution hypothesis, the data tells us:

- **Dominant-leg MM is exactly `L_pv_mark * 0.00715`** (no deduction,
  no hedge discount). This pins LTCUSDT first-tier `mmr_rate = 0.00715`.
- **Smaller-leg `position_im == position_mm == 0.14782244`** is
  invariant across mark changes. The smaller leg's published values
  do not move with mark in this hedged regime ‚Äî pointing at a rule
  whose smaller-leg term depends on entry-PV or on the dominant-leg
  state, not on mark.
- **Dominant-leg IM offset is `0.23248911` USDT** above
  `L_pv_mark / leverage = L_pv_mark * 0.1`. Constant across all 5
  rows. Likely related to either a tier-specific IM deduction or to
  the smaller-leg residual (its origin is not derivable from a
  constant-size dataset).
