# Feature 0096 — Migrate RULES.md to native path-scoped `.claude/rules/`

## Problem

`RULES.md` (1407 lines, ~15–18k tokens) is loaded manually per the workflow ("read
./RULES.md" in step 2): the whole file every time, regardless of task relevance, and
the content is lost on context compaction (never re-read). ~85% of the file is
component-scoped (gridcore, gridbot, replay, …) and only loads usefully when working
on that component.

Claude Code native rules (`.claude/rules/*.md` with a `paths:` frontmatter field)
load automatically only when Claude reads files matching the glob, and are
re-injected after compaction. Unscoped rules (no `paths:`) behave like CLAUDE.md
(always loaded).

## Goal / success criteria

1. Component sections of RULES.md live in per-component path-scoped rule files;
   cross-cutting content lives in one small unscoped rule file.
2. Zero content loss: every section of the original RULES.md exists verbatim in
   exactly one destination file (split is cut-paste, no rewording).
3. All rule files are **git-tracked** (currently the whole `.claude/` tree is
   gitignored and untracked — including the existing `code-style.md`).
4. `RULES.md` remains as a thin index (title + map of rule files) so that
   ~10 docstring/comment references to "RULES.md" in source files and the external
   review tooling (codex/cursor prompts) stay valid.
5. Verified live: fresh session starts without component rules in context; reading a
   `packages/gridcore/**` file injects the gridcore rule.

## Key facts discovered (research)

- `.gitignore:42` ignores `.claude/` (whole directory). `git ls-files .claude/` is
  empty — nothing tracked, `code-style.md` exists only locally. Git cannot re-include
  files under an excluded *directory* pattern; `.claude/` must become `.claude/*`
  before `!.claude/rules/` can work.
- Only ONE "entry N" cross-reference in RULES.md (line 1306: pitfall 19 → entry 16),
  both inside "Common Pitfalls" — they move together into one file; no renumbering.
- Files referencing "RULES.md" that need review: `.claude/commands/adopt-bot-pr.md`,
  `~/.claude/skills/plan-debate/planner.md` (user-level). The ext-plan-review /
  ext-code-review skills do NOT reference RULES.md directly. Source-code docstrings
  ("see RULES.md") stay valid via the thin index — no code changes.
- `grid-bot-validation/CLAUDE.md` does not mention RULES.md; the parent
  `PROJ/CLAUDE.md` workflow (steps 2 & 6) does, generically ("if present") — it
  covers other projects too, so it is edited minimally or not at all (user call).

## Phases

### Phase 0 — git plumbing (prerequisite)

1. Branch: `feature/0096-native-rules` in a separate worktree (tracked files:
   `.gitignore`, `RULES.md`, new `.claude/rules/*`).
2. `.gitignore`: change line 42 `.claude/` → `.claude/*`, add `!.claude/rules/`.
3. Verify with `git check-ignore -v`: `.claude/rules/*` NOT ignored;
   `.claude/settings.local.json`, `.claude/skills/`, `.claude/commands/` STILL
   ignored (nothing else accidentally becomes trackable).
4. `git add .claude/rules/code-style.md` — bring the existing rule under version
   control (first time).

### Phase 1 — pilot (verify the mechanism before bulk work)

1. Create ONE path-scoped rule, `.claude/rules/replay.md` (frontmatter: `paths:`
   YAML list of quoted globs, per code.claude.com/docs/en/memory "Path-specific
   rules"), cutting lines 1185–1232 out of RULES.md.
2. Manual test in a fresh session: rule absent at start; `Read` an
   `apps/replay/**` file → rule content appears in context. If the trigger
   semantics differ from expectation (e.g. glob form, list vs scalar), fix the
   format here — before creating the other 12 files.

### Phase 2 — bulk split

1. Execute the mapping table: pure cut-paste into the remaining rule files, each
   with its `paths:` frontmatter (format proven in Phase 1). No rewording, no
   cleanup — content sweep stays a separate future task (~feature 0100).
2. Rewrite `RULES.md` as the thin index.
3. Completeness audit (mechanical): every `##`/`###` header of the original file
   must appear in exactly one new location; line-count of (index + all new files)
   ≈ original minus dropped ToC.

### Phase 3 — reference updates

1. `.claude/rules/code-style.md`: "don't duplicate RULES.md" anti-pattern line →
   point at `.claude/rules/` + index.
2. `.claude/commands/adopt-bot-pr.md` (gitignored, edit in place): RULES.md
   pointer → mention `.claude/rules/` too.
3. `~/.claude/skills/plan-debate/planner.md` (user tree, edit in place): planner
   should read `.claude/rules/*.md` in addition to the RULES.md index.
4. `grid-bot-validation/CLAUDE.md`: add one line — component rules auto-load from
   `.claude/rules/`; "update RULES.md" workflow step now means updating the
   relevant rule file.
5. Parent `PROJ/CLAUDE.md` steps 2/6: leave generic wording ("read the project's
   ./RULES.md (if present)") — still correct via the index; optional broader edit
   is out of scope for this repo's PR.

### Phase 4 — verification

1. Fresh-session smoke test (repeat of pilot, post-split): no component rules at
   start; touch gridcore file → `gridcore.md` injected; touch nothing → only
   `code-style.md` + `core-invariants.md` present.
2. `git ls-files .claude/rules/` lists all 14 files (13 new + code-style.md).
3. `make lint` unaffected (markdown not linted) — run anyway as regression guard.
4. Header-completeness audit from Phase 2.3 passes.

## What is explicitly OUT of scope

- Content rewrite/pruning of any rule text (deferred to the planned ~0100 sweep).
- Changes to source-code docstrings mentioning RULES.md (still valid via index).
- Changes to ext-plan-review / ext-code-review skills (no RULES.md references).

## Risks

| Risk | Mitigation |
|---|---|
| `paths:` frontmatter syntax differs from assumption | Phase 1 pilot proves format on one file before bulk |
| gitignore negation silently fails (`.claude/` dir-level exclude) | `.claude/` → `.claude/*` change + `git check-ignore -v` verification in Phase 0 |
| Content lost during 13-way split | Cut-paste only + mechanical header/line audit in Phase 2.3 |
| Rules not loading for grep-only research (path rules trigger on Read) | Accepted: Read-before-Edit is mandatory anyway; core invariants are unscoped |
| External reviewers (codex/cursor) miss component rules | RULES.md index explicitly points them at `.claude/rules/`; plan-debate planner updated |

## Rollback

Single revert of the feature commit restores monolithic RULES.md and the old
`.gitignore`; `.claude/rules/*.md` deletions ride the same revert.
