# 0065 — Backtest wallet seed: re-mark non-USDT collateral (totalEquity / liq_price parity)

## Context

Issue #155. Replay seeds a single-coin/single-snapshot `WalletSeed` and the
backtest then evolves equity purely from futures fills. When the trader holds
non-USDT spot (e.g. SOL) as UTA collateral, live `totalEquity` floats with
that coin's mark while backtest stays anchored to the seed snapshot. The
systematic drift surfaces in liquidation-price and IM/MM-derived parity
metrics (`liq_price_*_abs_delta`, `position_im_*_abs_delta`,
`position_mm_*_abs_delta`). Per-trade PnL / fee / qty / `position_value` are
unaffected and must stay byte-parity.

Goal: track each non-USDT collateral coin's mark over the replay window and
add a collateral re-mark delta to `BacktestSession.total_equity` so it floats
like live **account `totalEquity`** (Bybit: USD sum of per-asset equity
**without** collateral value ratio / haircut). The futures `pnl_delta` path
still drives `current_balance` and `equity_curve` (available-balance
semantics, 0042). Add comparator metrics for attribution and for coins that
could not be modelled.

Verbatim from the spec (equity re-mark, **no haircut on this term**):
`totalEquity_t ≈ USDT_component_t + Σ_coin (coin_balance × mark_t)` where the
backtest implements only the **delta** from seed:
`Σ_coin balance × (mark_t − seed_mark)` added to `session.total_equity`
(anchored so t0 contribution is zero). **Do not** multiply this term by
collateral value ratio — that ratio applies to **margin balance**, not
`totalEquity` ([Bybit UTA glossary](https://www.bybit.com/en/help-center/article/Glossary-Unified-Trading-Account/),
[V5 wallet balance](https://bybit-exchange.github.io/docs/v5/account/wallet-balance)).
Optional seed-locked `collateral_value_ratios` may be stored for a **future**
margin-balance / IM parity follow-up; they are **out of scope** for Phase 2A
and acceptance #3a.

acceptance #3a requires end-of-window `totalEquity` parity between live
(account-level `total_equity` from `wallet_snapshots`) and backtest
(`session.total_equity`) within $1; acceptance #3b adds
`non_usdt_collateral_drift_total` as the backtest collateral re-mark term
`(collateral_now − seed_contrib)` for attribution (not the parity gap —
after Phase 2A that term is already inside `session.total_equity`, so a
correct model drives the gap in #3a toward ~$0); acceptance #4 requires
zero behaviour change when only USDT is held.

Out of scope (verbatim): funding for non-USDT coins; treating spot SOL as a
tradable backtest position; per-snapshot recomputation of Bybit
`accountIMRate`.

GROUNDING PREREQUISITE (blocking, do before Phase 1B implementation):
Confirm against a REAL recorded `wallet_snapshots.raw_json` row (or Bybit V5
UTA docs) per-coin wallet fields and whether they appear for a quiet
(unchanged) non-USDT collateral coin. Bybit's UTA wallet WS frequently pushes
only the coins that CHANGED, so a SOL balance that sits unchanged near `at_ts`
may not have a fresh per-coin row — `get_all_coins_latest_before` then returns
a stale row (acceptable for **balance** only) OR no row. Stale per-coin rows
must **not** supply `usdValue` for `seed_mark` (see valuation basis below) —
use ticker mark at `at_ts` instead to avoid a false jump on the first tick
when `initial_equity` comes from a fresh account-level USDT row.

Separate two concerns (do **not** conflate them):

1. **Inclusion for `totalEquity` re-mark** — **not** gated on collateral
   switches. Bybit `totalEquity` is the USD sum of per-asset equity regardless
   of whether the asset is enabled as margin collateral. Inclusion rule:
   `coin ∈ seed.collateral_coins` **and** a wallet row exists with
   `wallet_balance > 0`. Missing row → `collateral_excluded_coins` (cannot
   model). V5 `collateralSwitch` / `marginCollateral` are **booleans**
   (verified API docs) meaning “usable as margin collateral” — record on
   `WalletSeed.collateral_switch_off_coins` and log WARNING when false/missing
   while the coin is still modelled; **do not** drop the coin from the re-mark
   set for that reason alone.

2. **Collateral value ratio** (margin balance only — **not** used in Phase 2A
   `total_equity`): tiered ratio from REST `get-collateral-info` or operator
   `seed.collateral_value_ratios` (optional seed snapshot for a future margin
   parity feature). A single static ratio per coin is a coarse approximation
   if tier size matters; document that limitation. **0065 does not apply
   ratios to `total_equity` or acceptance #3a.**

### 1B. Multi-coin wallet seed dataclass + loader

File: `apps/replay/src/replay/snapshot_loader.py`.

- Extend `WalletSeed` (keep existing scalar fields for back-compat) with **six**
  new dict/list fields (all `field(default_factory=...)`) so existing
  constructors/tests don't break:
  `WalletSeed` is `@dataclass(frozen=True)` (verified: snapshot_loader.py:113–114;
  line 72 is `GridStateSeed`). Two consequences the plan MUST honour:
  (a) mutable `dict` defaults cannot be written as bare `= {}` — that raises
  `ValueError: mutable default <class 'dict'> ... is not allowed` at
  class-definition time; use `field(default_factory=dict)`. (b) Frozen means
  the loader CANNOT mutate an existing `WalletSeed`; it must construct a new
  instance with all fields in one `WalletSeed(...)` call (or
  `dataclasses.replace(existing, coin_balances=..., ...)`).
  - `coin_balances: dict[str, Decimal] = field(default_factory=dict)` —
    per-coin balance for every collateral coin (excludes the
    USDT/`wallet_coin` row, which is already `coin_balance`). Populate from
    the typed `WalletSnapshot.wallet_balance` Decimal column (models.py:497,
    `Numeric(20,8)`, already coerced via
    `decimal_or_zero(coin_data.get('walletBalance'))` by both the WS writer
    wallet_writer.py:306-308 and the REST path recorder.py:408-410) — do NOT
    re-parse the string `walletBalance` out of `raw_json`, to stay consistent
    with how `coin_balance`/`total_*` are read elsewhere in this loader.
  - `seed_marks: dict[str, Decimal] = field(default_factory=dict)` — each
    collateral coin's `*USDT` mark at-or-before `at_ts` (the t0 mark used to
    verify the re-mark term starts at zero net contribution; see Phase 2
    anchoring).
  - `collateral_value_ratios: dict[str, Decimal] = field(default_factory=dict)` —
    optional per-coin collateral value ratio locked at seed (from
    `get-collateral-info` or `SeedConfig.collateral_value_ratios`). **Not**
    applied to `total_equity` in 0065 — stored for a future margin-balance
    parity follow-up only.
  - `collateral_excluded_coins: list[str] = field(default_factory=list)` —
    configured coin with no wallet row (or zero balance) at seed time.
  - `collateral_missing_mark_coins: list[str] = field(default_factory=list)` —
    configured coin with a balance row but no seed mark resolved (absent from
    `coin_balances` / `seed_marks` / `collateral_value_ratios` in the loader
    return).
  - `collateral_switch_off_coins: list[str] = field(default_factory=list)` —
    coin is modelled for `totalEquity` but `collateralSwitch` /
    `marginCollateral` was false or missing in `raw_json` (metadata / WARN
    only; does not remove the coin from re-marking).
- Add `WalletSnapshotRepository.get_all_coins_latest_before(run_id,
  account_id, at_ts)` in `shared/db/src/grid_db/repositories.py`: returns the
  latest row per `coin` at-or-before `at_ts` (group-by-coin, max
  `exchange_ts`). Used by `load_collateral_seed` to find per-coin rows (balances
  committed only after seed mark resolves).
- Add `TickerSnapshotRepository.get_mark_at_or_before(symbol, at_ts) ->
  Optional[Decimal]` in `shared/db/src/grid_db/repositories.py`: latest
  `mark_price` where `exchange_ts <= at_ts` (same carry-forward semantics
  as wallet seed — if no row exists at-or-before `at_ts`, return `None`).
  `get_latest_by_symbol` is "latest overall" and is wrong for seed-time
  lookup; used by `CollateralMarkFeed.mark_at` (Phase 2B) for **intra-run**
  mark updates. Tests in `shared/db/tests/test_repositories.py`.

**Valuation basis (seed vs intra-run):** Bybit account `totalEquity` uses
Bybit’s per-asset USD valuation (`usdValue` on the per-coin wallet object in
V5 payloads), which can differ from a configured `*USDT` perp
`TickerSnapshot.mark_price` (index basis, timing, basis risk).

**Seed anchor freshness (load-bearing):** `initial_equity` comes from the
latest account-level USDT wallet row near `at_ts`. A quiet collateral coin may
have a **much older** per-coin row; its `usdValue` is then stale while
`initial_equity` is current. Using stale `usdValue / wallet_balance` as
`seed_mark` makes the first ticker update add
`balance × (current_mark − stale_seed_mark)` on top of an already-current
`initial_equity` → false equity jump. Therefore:
- Use `seed_mark = usdValue / wallet_balance` **only when** the coin row is
  fresh: `_strip_tz(at_ts) − _strip_tz(row.exchange_ts) <= wallet_max_staleness`
  (add to `SeedConfig`, default **60s**) **and** both keys exist. **`_strip_tz`**
  matches `engine._seed_pre_check` (engine.py:766–767): SQLite may return naive
  `exchange_ts` while config `at_ts` is aware — strip tz uniformly before
  subtracting or the loader raises `TypeError`.
- Otherwise use `get_mark_at_or_before(symbol, at_ts)` for the seed anchor
  (balance may still come from the stale row — mark path stays consistent with
  ticker-driven intra-run updates). Log WARNING when falling back due to
  staleness.

Intra-run re-marking uses ticker marks only (`balance × (mark_t − seed_mark)`).
Risks: perp mark vs wallet valuation divergence may miss tight #3a on large
balances; optional follow-up: end-of-window per-coin `usdValue` at `end_ts`.
- New loader `load_collateral_seed(db_session, run_id, account_id, at_ts,
  usdt_coin, collateral_coins: list[str],
  collateral_symbol_for: dict[str, str], configured_value_ratios:
  dict[str, Decimal], wallet_max_staleness: timedelta) -> tuple[...]`:
  (`wallet_max_staleness` from `seed.collateral_wallet_max_staleness`).
  At module top or inside the loader, reuse the same `_strip_tz(dt)` helper as
  `engine._seed_pre_check` for every `at_ts` vs `row.exchange_ts` comparison.
  1. Calls `get_all_coins_latest_before`.
  2. Initialise `excluded_coins: list[str] = []`, `missing_mark_coins: list[str] = []`,
     `switch_off_coins: list[str] = []`.
  3. For each `coin` in `collateral_coins` (operator list — do not iterate
     every DB coin): find the latest row in `get_all_coins_latest_before` for
     that coin. If no row or `wallet_balance == 0`, append to
     `excluded_coins`, WARNING, skip (do not touch model dicts). Otherwise
     proceed to step 4 for that coin (balance row held in loop-local `row` only
     until a seed mark is resolved).
  4. **Per coin with a balance row** — resolve `seed_marks[coin]` first; only
     on success commit the coin into the returned model dicts:
     - **Preferred (fresh row only):** `usdValue / wallet_balance` when
       `usdValue` is present, balance > 0, and
       `_strip_tz(at_ts) − _strip_tz(row.exchange_ts) <= wallet_max_staleness`
       (default 60s — coin row co-temporal with `at_ts`, not a quiet-coin stale
       push; tz-normalized per engine pre-check pattern).
     - **Otherwise:** `get_mark_at_or_before(collateral_symbol_for[coin], at_ts)`
       (required when row is stale even if `usdValue` exists — WARNING
       `stale wallet row for seed mark, using ticker`).
     - **On success:** set `seed_marks[coin]`, `coin_balances[coin] =
       row.wallet_balance`, and optionally `collateral_value_ratios[coin]` from
       config. Read `collateralSwitch` / `marginCollateral` as **booleans** for
       metadata only: when false/missing, append to `switch_off_coins` and
       WARNING — coin remains in all three model dicts.
     - **On failure** (no usable seed mark): append to `missing_mark_coins`,
       WARNING, and **do not** add the coin to `coin_balances`, `seed_marks`, or
       `collateral_value_ratios` (equivalent: if step 3 had tentatively inserted
       entries, `pop` them before continuing). Returned dicts must contain only
       fully modelled coins — `BacktestSession` and #3b must not see a balance
       without a matching `seed_marks[coin]`. Live `totalEquity` still includes
       the asset; #3a gap may appear; export lists surface it.
  5. Returns `(coin_balances, seed_marks, collateral_value_ratios,
     excluded_coins, missing_mark_coins, switch_off_coins)`.
  6. **`engine._load_seed` merge (required):** after `load_wallet_seed_full`,
     when `seed.collateral_coins` is non-empty:
     - If `wallet_seed is None`, **raise `SeedDataQualityError`** with a message
       that collateral re-marking requires a valid account-level wallet seed
       (`total_equity` / `total_available_balance` > 0 at `at_ts`). Do **not**
       log the existing soft-fallback WARNING and continue — `dataclasses.replace`
       would fail on `None`, and `initial_equity` would not match live anyway.
     - Else call `load_collateral_seed` in the same DB session and
       `wallet_seed = dataclasses.replace(wallet_seed, coin_balances=...,
       seed_marks=..., collateral_value_ratios=...,
       collateral_excluded_coins=excluded, collateral_missing_mark_coins=missing,
       collateral_switch_off_coins=switch_off)`.
     When `collateral_coins` is empty, keep today's soft-fallback on missing
     wallet seed unchanged. Do **not** extend the `_load_seed` return tuple.
     File impact: `engine.py::_load_seed` (~699–720) — gate before
     `return wallet_seed, ...` and before the post-session WARNING block.
- Coin→symbol mapping: derive `f"{coin}USDT"` by default; allow override via
  a config map (see Phase 1C). USDC is pegged — spec says "cheap to track";
  default mapping `USDC → USDCUSDT`.

## Phase 2 — Backtest equity re-marking

File: `apps/backtest/src/backtest/session.py`, `apps/replay/src/replay/engine.py`.

### 2A. Session — collateral re-mark term

- `BacktestSession.__init__` gains optional collateral state:
  `collateral_balances: dict[str, Decimal] = {}`,
  `collateral_seed_marks: dict[str, Decimal] = {}` (modelled coins only).
  Store them; compute the per-coin seed contribution once:
  `seed_contrib = Σ balance × seed_mark` (**no** collateral value ratio).
- Add `self.collateral_marks: dict[str, Decimal]` initialised to
  `collateral_seed_marks` (latest mark seen per coin).
- New method `update_collateral_mark(coin: str, mark: Decimal)` — sets
  `collateral_marks[coin] = mark` for coins in the collateral set (no-op
  otherwise). Cheap; called from the engine loop when a collateral coin's
  ticker arrives.
- Equity formula change (the ONLY behavioural change): in BOTH
  `update_equity` (session.py:257) AND `refresh_balances` (session.py:225) —
  the two sites that recompute `total_equity` with the IDENTICAL `pnl_delta`
  formula — after computing `pnl_delta`, add a collateral re-mark DELTA to
  `total_equity` ONLY. `refresh_balances` MUST get the term too (not just
  `update_equity`): it is the site `process_fills` calls (runner.py:545)
  immediately before reading `total_equity` for `_estimate_pair_liq_prices`
  and the emitted snapshot, so omitting it there would leave the paired
  acceptance-#1 metrics on the seed (un-re-marked) equity.
  - `collateral_now = Σ balance × collateral_marks[coin]` (full asset USD
    value at mark — matches Bybit `totalEquity` component, not margin balance)
  - `total_equity = initial_equity + pnl_delta + (collateral_now − seed_contrib)`
  - `current_balance` (available balance) is unchanged — it stays
    `initial_balance + pnl_delta`. **`equity_curve`** continues to append
    `(timestamp, initial_balance + pnl_delta)` — the same value as
    `current_balance`, **not** `total_equity` (see session.py:253–257 today).
    Collateral revaluation moves `total_equity` only, keeping executor
    sizing / qty / margin gating byte-identical (preserves PnL/fee/qty
    parity, spec "NOT affected" list).
  - When the collateral set is empty, `collateral_now == seed_contrib == 0`,
    so `total_equity` is unchanged → acceptance #4 (USDT-only no-op).
  - `finalize()` `final_balance` is available-based (unchanged); it must NOT
    pick up the collateral term (final_balance feeds return/turnover, which
    are available-denominated). Document this explicitly.

Anchoring rationale: subtracting `seed_contrib` makes the term a pure delta
from t0, so the seed snapshot stays consistent with `initial_equity`
(`total_equity` at t0 == live `totalEquity` at `at_ts`). The drift is purely
`balance × (mark_t − seed_mark)` (no ratio on this term).

### 2B. Engine — feed collateral marks per tick

File: `apps/replay/src/replay/engine.py`.

- Build the `BacktestSession` with collateral kwargs from
  `wallet_seed.coin_balances / seed_marks` (empty when no seed / no
  collateral). After `session.finalize()`, copy
  `wallet_seed.collateral_excluded_coins`, `wallet_seed.collateral_missing_mark_coins`,
  and `wallet_seed.collateral_switch_off_coins` into `ValidationMetrics`
  (Phase 3) — lists must survive on `WalletSeed`, not a separate `_load_seed`
  return value.
- The traded-symbol `HistoricalDataProvider` does not carry collateral coin
  ticks. Add a collateral mark source:
  - New helper (in `engine.py` or `data_provider.py`) that, per collateral
    coin, streams `(exchange_ts, mark_price)` from `ticker_snapshots` for the
    coin's symbol across `[start_ts, end_ts]`, and the engine merges these
    into the main loop by timestamp. Simplest correct approach: before/at
    each traded tick, advance a per-coin cursor and call
    `session.update_collateral_mark(coin, latest_mark_at_or_before(tick_ts))`.
    Implement as a small `CollateralMarkFeed` that holds one ordered
    `TickerSnapshot.mark_price` list per coin (reuse the cursor-pagination
    pattern from `HistoricalDataProvider._iterate_tickers`) and exposes
    `mark_at(coin, ts)`.
  - Call `session.update_collateral_mark(...)` for each collateral coin at
    the TOP of the tick loop — BEFORE `runner.process_fills(tick)`
    (engine.py:305), not merely before `session.update_equity`
    (engine.py:312). This is load-bearing: the ONLY backtest-side source of
    the `liq_price`/`position_im`/`position_mm` metrics the comparator pairs
    (acceptance #1) is `_emit_position_snapshot`, which runs INSIDE
    `process_fills` (runner.py:607). On a fill tick, `process_fills` calls
    `refresh_balances` (runner.py:545) then
    `_estimate_pair_liq_prices(..., self._session.total_equity, ...)`
    (runner.py:551-554) to compute the emitted snapshot's `liq_price`. If the
    collateral mark were refreshed only after `process_fills` returns, the
    emitted backtest liq/IM/MM on every fill row would use the PREVIOUS tick's
    collateral mark — lagging the very metric this feature converges by one
    collateral step on exactly the rows the comparator measures. Updating the
    mark at loop top fixes this.     `mark_at(coin, ts)` uses the same at-or-before semantics as
    `get_mark_at_or_before` (carry-forward the latest mark with
    `exchange_ts <= ts`) and must be queried with the current
    `tick.exchange_ts` and applied before any fill/snapshot/equity work for
    that tick.
  - In `_wind_down` / `finalize`, the last-seen collateral marks already sit
    in `session.collateral_marks`; no extra collateral-feed work needed.
    **Wind-down equity refresh (load-bearing for #3a):** today
    `engine._wind_down` (`close_all`) calls `tracker.process_fill` +
    `session.record_trade` but does **not** call `refresh_balances` or
    `update_equity`, and `session.finalize()` does not rewrite
    `session.total_equity`. So after `run()`, `session.total_equity` still
    reflects the last tick-loop `update_equity` (open-position unrealized),
    not the post-close state. Phase 2 must pick one fix:
    - **(preferred for #3a integration test)** assert acceptance #3a only
      with `wind_down_mode: leave_open` (default in replay conf examples);
      document that `close_all` is out of scope for the #3a parity check, OR
    - capture `backtest_total_equity` immediately after the tick loop, before
      `_wind_down`, in the test harness / `verify_0065_collateral.py`, OR
    - extend `_wind_down` to call `session.refresh_balances(unrealized=0)`
      (and apply the Phase 2A collateral term there too) after each forced
      close so `total_equity` matches flat positions at `end_ts`.

Memory note: collateral feeds are bounded by ticker volume for the extra
symbols (spec risk: a few hundred MB for a 1+ month window). Stream with the
existing batch cursor; do not materialise all marks eagerly for very long
windows — page on demand keyed by the traded-tick timestamp.

### Acceptance (end-of-window checks)

  `abs(live_total_equity(end_ts) − backtest_total_equity(end)) < $1`.

  `wallet_snapshots` row at-or-before `end_ts` (see algorithm below).
  Backtest value: `session.total_equity` at end-of-window — **not** the
  stale value left by `close_all` wind-down (see Phase 2B wind-down note).
  Read it either (a) immediately after the tick loop, before `_wind_down`, or
  (b) after `_wind_down` only when wind-down refreshes balances (Phase 2B
  option). Do **not** read post-`finalize()` expecting a refresh;
  `finalize()` does not update `session.total_equity`. Integration tests
  MUST use `wind_down_mode: leave_open` unless the wind-down refresh fix
  is implemented.
  **Static-balance limitation:** `coin_balances` are locked at seed
  (`at_ts`); live `totalEquity` can drift if the trader deposits, withdraws,
  or trades the collateral coin during the window. #3a measures mark re-mark
  parity (balance × Δmark), not spot-balance drift — document WARN when live
  coin balance at `end_ts` differs materially from seed.

## Notes / risks (verbatim alignment)

- **Bybit semantics:** `totalEquity` excludes collateral value ratio;
  margin balance / available balance use ratio and haircut loss. 0065 re-marks
  `total_equity` only (no ratio on the delta). Optional `collateral_value_ratios`
  on `WalletSeed` are for a future margin-balance feature, not acceptance #3a.
- **Inclusion:** `collateral_coins` + wallet row + balance > 0 only; boolean
  switches are WARN metadata (`collateral_switch_off_coins`), not exclusion.
- **Valuation basis:** seed anchor uses `usdValue`/`balance` only when the
  coin wallet row is fresh vs `at_ts` (default 60s, `_strip_tz` on both sides);
  stale rows use ticker at `at_ts`. Intra-run uses perp `mark_price`.
- **Wallet seed required:** non-empty `collateral_coins` with `wallet_seed is
  None` → `SeedDataQualityError` (no soft-fallback).
- **Under-coverage:** `collateral_missing_mark_coins` / `collateral_excluded_coins`
  omit coins from #3b; check CSV list rows and #3a gap.
- Static collateral balances: `coin_balances` are frozen at seed time; live
  spot balance changes during the window are not modelled (#3a risk — see
  acceptance #3a limitation).
- Coin-set determination: static via `seed.collateral_coins` +
  `recorder.collateral_symbols` (documented "configure all coins ever held").
- Ticker volume growth: accepted; only the configured collateral symbols are
  added to the public subscription.
- `accountIMRate` exact match is explicitly out of scope; this plan improves
  the `totalEquity`-driven inputs only.
